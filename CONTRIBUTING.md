# CONTRIBUTING

Спасибо за интерес к репозиторию со skill-ами.

Этот репозиторий предназначен для хранения и развития skill-ов для Codex. Новые skill-ы и улучшения существующих принимаются через Pull Request.

## Как предложить новый skill

1. Сделайте `fork` репозитория.
2. Создайте отдельную папку для нового skill.
3. Добавьте обязательные файлы.
4. Обновите корневой `README.md`, добавив короткую строку в каталог.
5. Откройте Pull Request с кратким описанием, что именно добавлено.

## Ожидаемая структура

Для каждого skill ожидается отдельная папка:

```text
skill-name/
  SKILL.md
  agents/openai.yaml
  references/
```

Минимально обязателен файл:

- `SKILL.md`

Дополнительно можно добавлять:

- `agents/openai.yaml`
- `references/`
- другие файлы, если они действительно нужны skill-у

## Требования к skill

- Один skill должен решать одну понятную задачу.
- Название папки должно быть коротким, в нижнем регистре, через дефис.
- Описание должно быть конкретным и понятным без лишнего текста.
- Skill не должен содержать лишнюю документацию, не относящуюся к его использованию.
- Если skill требует внешние сервисы или особый контекст, это должно быть явно указано в `SKILL.md`.

## Требования к каталогу

При добавлении нового skill обновите корневой `README.md`.

Формат записи:

```md
- [`skill-name`](./skill-name) — короткое описание того, что делает skill.
```

Описание должно быть коротким: одна строка, без примеров и без длинных пояснений.

## Что желательно указать в Pull Request

- зачем нужен skill;
- какие задачи он решает;
- есть ли особенности установки или использования;
- есть ли ограничения или зависимости.

## Чего делать не нужно

- не складывайте несколько разных skill-ов в одну папку;
- не добавляйте большие вспомогательные файлы без необходимости;
- не дублируйте одно и то же описание в нескольких местах;
- не перегружайте каталог длинными инструкциями.

## Проверка перед отправкой

Перед созданием Pull Request убедитесь, что:

- структура папки аккуратная;
- `SKILL.md` заполнен;
- название skill и описание в каталоге согласованы;
- в репозиторий не попали лишние временные или служебные файлы.
